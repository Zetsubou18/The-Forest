Техническое описание проекта "The Forest"

Обзор проекта:
"The Forest" — хоррор-игра с AAA-качеством графики и текстур, созданная в Unity). 
Игрок исследует лес, используя фонарик, изучает точки интереса, обозначенные на карте, и избегает монстра. 
Проект акцентирует внимание на пугающей атмосфере, создаваемой звуковыми эффектами, скримерами и высококачественной графикой.

Техническая реализация:
- Версия Unity: 2022.3.37f1
- Язык программирования: C#
- Игровая механика:
  - Исследование: Игрок перемещается по лесу, освещая путь фонариком (реализовано через Spotlight с управлением мышью/клавиатурой).
  - Карта: Интерактивная карта (открывается клавишей "M") с подсвеченными точками интереса, созданная с использованием Unity UI и скриптов для отображения маркеров.
  - Монстр: Управляется системой NavMesh для навигации по локации. Периодически издаёт крик (аудиоклип с таймером) для индикации местоположения.
  - Индикатор сердцебиения: Активируется при близости монстра, реализовано через расчёт расстояния между игроком и монстром с помощью скрипта (Vector3.Distance).
  - Скримеры: Триггерные зоны в определённых локациях, запускающие анимации и звуковые эффекты.
- Графика: Использованы высококачественные текстуры и шейдеры для создания реалистичного леса. Локация построена с использованием системы Terrain в Unity.
- Аудио: Хоррор-музыка и звуковые эффекты (шорохи, крики монстра) для усиления атмосферы, интегрированы через AudioSource.
- Интерфейс: UI для карты и индикатора сердцебиения, реализован с использованием Canvas и скриптов для динамического обновления.

Проблемы, с которыми столкнулся:
- Проблема с NavMesh и деревьями: NavMesh в Unity не распознавал деревья на Terrain как препятствия, из-за чего монстр строил маршруты 
через них, но не мог пройти из-за коллайдеров. Решено отключением коллайдеров деревьев, где монстр проходит через деревья.
- Оптимизация: Высокая нагрузка на рендеринг из-за AAA-текстур и большого количества объектов. Решено с помощью:
  - Occlusion Culling: Рендеринг только видимых игроком объектов.
  - Переписывание скриптов для минимизации вычислений (например, замена триггерной зоны для сердцебиения на расчёт расстояния через Vector3.Distance).
  - Снижение качества теней и LOD для дальних объектов.
- Триггерная зона для сердцебиения: Изначально использовалась большая триггерная зона, что было неэффективно. Заменено на расчёт расстояния между игроком и монстром, что улучшило производительность.

Уроки, вынесенные из проекта:
- Работа с NavMesh в Unity требует тщательной настройки объектов Terrain, чтобы избежать проблем с навигацией ИИ.
- Оптимизация AAA-графики требует комплексного подхода (Occlusion Culling, LOD, оптимизация скриптов).
- Точные расчёты расстояний более эффективны, чем большие триггерные зоны, для динамических механик.

Возможные улучшения:
- Исправить баг с NavMesh, добавив кастомные препятствия для деревьев или используя альтернативную систему навигации.
- Добавить больше точек интереса или случайных событий для повышения реиграбельности.
- Улучшить скримеры с помощью анимаций и дополнительных визуальных эффектов.
- Оптимизировать текстуры и шейдеры для дальнейшего повышения FPS на слабых устройствах.
